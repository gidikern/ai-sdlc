# PRD Template

## Document Header

```markdown
# Product Requirements Document: [Product Name]

**Version**: 1.0
**Date**: YYYY-MM-DD
**Author**: [Name]
**Status**: Draft | Review | Approved
```

## Sections

### 1. Executive Summary
2-3 sentences describing what we're building and why.

### 2. Problem Statement
- What problem are we solving?
- Who has this problem?
- How are they solving it today?
- Why is the current solution inadequate?

### 3. Goals & Success Metrics

| Goal | Metric | Target |
|------|--------|--------|
| Primary goal | How we measure | Success threshold |

### 4. Target Users

**Primary Persona**:
- Demographics
- Goals
- Pain points
- Current behavior

**Secondary Personas** (if applicable):
- Brief description

### 5. Solution Overview
High-level description of the proposed solution.

### 6. Scope

**In Scope (MVP)**:
- Feature 1
- Feature 2

**Out of Scope (Future)**:
- Feature A
- Feature B

**Non-Goals**:
- What we explicitly won't do

### 7. User Journeys
Key user flows with steps.

### 8. Requirements

#### Functional Requirements
| ID | Requirement | Priority |
|----|-------------|----------|
| FR-001 | Description | Must/Should/Could |

#### Non-Functional Requirements
- Performance: [targets]
- Security: [requirements]
- Scalability: [expectations]
- Accessibility: [standards]

### 9. Assumptions & Constraints
- Technical constraints
- Business constraints
- Dependencies

### 10. Risks & Mitigations

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Description | H/M/L | H/M/L | Strategy |

### 11. Timeline
High-level milestones (detailed planning in separate document).

### 12. Open Questions
Items needing resolution before development.
